# AI Usage Policy

Kan is built with a great amount of AI assistance, and we view it as a
productive tool in the right hands. This policy sets guidelines for
responsible AI usage with respect to Kan's development.

- **Open an issue before a sizeable PR.** For anything beyond a small,
  obvious fix, please open or comment on an issue first so we can get
  on the same page about the approach. It saves you from building
  something that ends up rejected, and it saves everyone time.

- **Changes must be human-tested.** Don't submit code that is only
  hypothetically correct. AI is great at producing plausible code
  that doesn't actually work. Run it, use it, and don't let AI write
  code for platforms or environments you can't manually test on
  yourself.

- **Keep AI-generated text tight.** AI loves to be verbose and pad
  things with noise that distracts from the point. If you use AI to
  help write an issue, discussion, or PR description, read it, edit it,
  and trim it down before posting. Text must have a human editor.

- **Use AI with care.** AI is a genuinely useful tool, but it has to be
  used with intention and respect for other people's time and effort.
  Contributors who fire off careless, unvalidated AI output aren't
  welcome, and repeat offenders will be banned. You've been warned.

These guidelines are for outside contributions. Maintainers use AI
tools at their discretion.

## There are Humans Here

Every issue, discussion, and pull request lands in front of a real
person. That's a point where you're asking for someone else's time and
attention. Showing up with low-effort, unvalidated work shifts the
burden of figuring out whether it's any good onto the maintainer, which
is a bad way to treat the person on the other side.

## AI is Welcome Here

Kan is written with plenty of AI assistance, and we embrace AI as a
productive part of the workflow. We love AI as a tool!

However, a few careless people can generate an enormous volume of
low-quality contributions, far more than they ever could by hand. The
cost of producing it is tiny; the cost of reviewing it is not.

We are not anti-AI; Kan wouldn't exist without it. The line is
between using AI with care and using it carelessly. Responsible AI
users are welcome here.
